# annual-project-musicalendar
Annual project for the Professional Practice 2 class of the Systems' Analyst curriculum

# OVERVIEW
Billboard and online promotion system for music platforms and live performance calendar.
Other possible names: Mar del Calendar, Hoy se sale, etc.
Examples:
https://www.alternativateatral.com/cartelera.asp
https://espaciocuatroelementos.com/section/346

# Requirements

# Non-functional:
Permitted technologies and languages: Node.js, Apache, MySQL, MongoDB. HTML, CSS, and JS, PHP (optional).
The system must not consume server resources due to saving video or image files.
Only link uploads are allowed.

# Functional:
The system must manage two user roles: Administrator and Artist.

# Administrator functions:
Validate that the artist is a real person.

# Artist functions:
CRUD of personal information: Artist photo (link to Google Drive), contact: email, phone.
CRUD of personal website or online portfolio.
CRUD of streaming platforms and others: Spotify, Apple Music, Tidal, YouTube Music, etc.
CRUD of YouTube channel.
CRUD of Instagram.

# Calendar function:
Date and time.
Venue of the performance.
Entry type: hat passing, free, benefit, paid.
Price (optional).
Link to online ticket sales platform (optional).
Link to flyer on Google Drive.

The system will manage a billboard that will be automatically generated based on the information artists enter into the live performance calendar.
